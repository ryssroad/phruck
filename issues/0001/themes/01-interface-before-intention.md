# Theme 01 — Interface Before Intention

## Question

What happens when the gesture reaches the interface before the thought finishes forming?

## Objects

- phone reach before remembering why;
- prompt box before problem definition;
- search before recall;
- feed before body check;
- notification before intention;
- agent summon before the operator decides whether help is needed.

## Mechanism

The interface becomes a pre-intention layer. It does not only answer an intention; it helps select which intention survives long enough to be felt.

Sequence:

1. Empty interval appears.
2. Gesture fills the interval.
3. Interface proposes the next object.
4. The proposal is experienced as convenience.
5. Repetition turns the proposal into a reflex.

## Human scene need

Geneva should find or construct one concrete scene where the first movement is not a decision but a reach.

Possible scene seeds:

- waking and touching the phone before knowing the need;
- opening a chat with an agent before naming the task;
- refreshing a dashboard after a meeting because silence feels unsafe.

## Source need

Public sources are optional for the first fragment, but a fuller piece should later connect this lane to attention, habit formation, interface defaults, or interruption research.

## Failure mode

Generic screen-time critique. The theme is not `phones are bad`; it is `the interface can arrive before intention`.
