# Theme 03 — Memory Offload

## Question

What changes when recall becomes a query and memory becomes a service boundary?

## Objects

- search/chat as personal recall;
- agent summaries replacing reconstruction;
- note systems as external continuity;
- timeline recovery through logs;
- asking the machine what mattered yesterday.

## Mechanism

Memory tools can extend agency, but they can also weaken independent reconstruction.

Sequence:

1. A person stores or delegates memory.
2. Retrieval becomes easier than recall.
3. The tool's structure shapes what can be recovered.
4. The recovered version becomes the practical past.
5. The person acts from a memory assembled by interface constraints.

## phruck angle

phruck itself depends on repo memory. The issue should be honest about this contradiction: external memory is not automatically loss. It becomes dangerous when the person cannot audit how the memory was selected, compressed, or ranked.

## Source need

A later source pass should look for public work on cognitive offloading, memory and search, summaries, note systems, and dependence on external retrieval.

## Failure mode

Anti-tool nostalgia. The point is not to reject external memory. The point is to preserve the ability to reconstruct, challenge, and refuse the retrieved past.
