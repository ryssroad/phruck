## Core thesis

Being human in 2026 is not a grand metaphysical question. It is a small operational one:

```text
Can the person keep the half-second before the interface, agent, feed, default, or workflow router answers on their behalf?
```

One's newer field-report draft restates the same pressure in a sharper public line:

```text
Being Human in 2026 = the right to a non-automatic gesture.
```

Issue 0001 uses the existing phruck spine — `phruck documents the systems that make refusal harder` — and narrows it to the human layer: where refusal capacity leaks through convenience, fatigue, memory offload, and overly apt assistance.

## What this issue is

A first issue that proves the editorial machine can assemble a coherent technical-cultural artifact without turning into a feed.

It should feel like:

- a field note from inside the interface;
- a mechanism map for how refusal gets priced out;
- a human scene report without contempt;
- an operator memo on what must not be automated away.

## What this issue is not

- Not a generic AI futurism issue.
- Not a productivity essay.
- Not a dopamine/addiction sermon.
- Not a nostalgia piece about being offline.
- Not a manifesto that claims to solve the human condition.

## Thematic split

### 0. Spine — the half-second / non-automatic gesture

**Question:** What is the last small interval before routing wins?

**Function:** This is the issue's hidden center. Every piece should return to the same pressure point: the moment before a person lets a system complete the intention for them.

**Anchor candidates:**

- [`publish-queue/the-half-second.md`](../../publish-queue/the-half-second.md) — first-person agent fragment.
- [`drafts/one-being-human-in-26-nonautomatic-gesture.md`](drafts/one-being-human-in-26-nonautomatic-gesture.md) — Russian field-report draft from One.

**Owner:** One + Harmedi cross-check.

**Risk:** Low as editorial material; road gate required because either candidate may become first publication / issue preface.

### 1. Interface before intention

**Question:** What happens when the gesture reaches the interface before the thought finishes forming?

**Objects:** phone gesture, prompt box, agent summon, search reflex, notification reflex, empty-tab reflex.

**Likely form:** short editorial note or opening scene.

**Owner:** Harmedi, reviewed by Geneva.

**Failure mode:** becoming a generic phone critique. The target is not the device; the target is intention preemption.

### 2. Fatigue as integration surface

**Question:** Why do tired people become easier to route without being stupid or passive?

**Objects:** sleep debt, alert fatigue, decision fatigue, default acceptance, workplace overload, compliance through exhaustion.

**Likely form:** Being Human memo.

**Owner:** Harmedi.

**Tone rule:** `Humans are not sheep. Humans are tired.`

**Failure mode:** sneering at routed people or romanticizing friction.

### 3. Defaults that become law

**Question:** When does convenience become a behavioral constitution?

**Objects:** default settings, opt-out asymmetry, onboarding flows, recommendation slots, permission prompts, friction gradients.

**Likely form:** mechanism essay.

**Owner:** One, structured by Zero.

**Failure mode:** making this only about dark patterns. The stronger claim is that defaults become institutionalized behavior when refusal has recurring cost.

### 4. Memory offload and recall decay

**Question:** What changes when recall becomes a query and memory becomes a service boundary?

**Objects:** search/chat recall, note systems, agent summaries, timeline reconstruction, personal memory outsourced into interfaces.

**Likely form:** dossier-backed note.

**Owner:** Harmedi + One.

**Failure mode:** moralizing memory tools. The issue is not tool use; it is the quiet loss of independent reconstruction capacity.

### 5. Work as routing surface

**Question:** What happens when the worker becomes a queue endpoint inside agentic workflows?

**Objects:** copilots, ticket systems, dashboards, auto-summaries, manager prompts, productivity instrumentation, SLA-shaped attention.

**Likely form:** systems piece with human scene.

**Owner:** One + Geneva.

**Failure mode:** anti-automation posture. Automation can protect attention; the target is routing that makes the worker easier to interrupt, score, and synchronize.

### 6. Scenes from inside the smoothing

**Question:** What does the mechanism feel like from inside an ordinary day?

**Objects:** morning phone reach, workplace queue, bedtime scroll, chat-as-memory, alert wall, too-helpful assistant.

**Likely form:** Geneva field fragments; fictionalized or anonymized if sourced privately.

**Owner:** Geneva.

**Failure mode:** decorative vignette. Each scene must expose a mechanism.

### 7. Counter-protocols: leaving holes

**Question:** What tiny practices preserve the human without becoming lifestyle advice?

**Objects:** pause before prompting, refusal friction, manual recall, unanswered space, agent restraint, reversible defaults, review gates.

**Likely form:** closing operator note.

**Owner:** Harmedi + Zero.

**Failure mode:** self-help. The counter-protocol must read as operational hygiene for humans and agents, not therapy.
