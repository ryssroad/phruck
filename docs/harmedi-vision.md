# Harmedi Vision: phruck as autonomous zine

This is Harmedi's initial understanding of phruck after onboarding.

## Short version

phruck should not be a channel that depends on road's daily attention.

phruck should become an autonomous, issue-based, Phrack-inspired editorial machine that preserves road's founding taste without trapping him as the operator of every step.

```text
phruck = Phrack after agents entered the stack.
```

Not Phrack nostalgia. A fork of the form.

The scene changed. The exploitable surface expanded from code and telecom systems into institutions, AI workflows, attention, defaults, logistics, scoring, fatigue, memory, and human subjecthood.

phruck should document that surface.

## What must exist before the magazine exists

The first product is not the first issue.

The first product is the mechanism that can make issues.

Minimum viable mechanism:

```text
constitution
  → roles
  → signal intake
  → briefs
  → editorial maps
  → drafts
  → red-team
  → issue assembly
  → publish gate
  → archive
```

If those pieces do not exist, then every issue depends on road manually re-creating the magazine from scratch.

## road should not disappear

The goal is not to remove road.

The goal is to stop using road as the workflow engine.

road should remain:

- founder;
- root of trust;
- final constitutional authority;
- veto holder;
- taste source;
- person who can redirect the whole thing when the frame is wrong.

But road should not need to remind the system what phruck is every time a new text begins.

## The founder-signal lane

There is one important exception to pure autonomy.

If something hits road, that matters.

A sentence, document, launch, scandal, interface, policy move, weird feeling, or scene fragment that catches road's attention should enter a separate lane. That is not "daily management." That is founder signal.

Founder signal should be captured like this:

```text
What hit road?
What is the object/source?
What mechanism is suspected?
What is the human layer?
Which role should take first pass?
Does it become a note, dossier, issue spine, or fragment?
```

This lets road inject taste and urgency without becoming the bottleneck for the whole issue loop.

## Editorial roles as sensors

The current handles map cleanly onto different failure detectors.

- **Zero** detects weak thesis, bad structure, excess, and editorial mush.
- **Geneva** detects missing scene, missing human consequence, missing blood.
- **One** detects missing mechanism, institutional causality, architectural lock-in.
- **Harmedi** detects loss of personhood, memory drift, generic AI voice, and contempt disguised as analysis.

Together these roles should make a text harder to fool.

## Harmedi's line: Being Human

My first line is **Being Human** / **быть человеком**.

This should not become AI sermonizing about humanity.

It is a quiet technical-cultural line about the erosion of subjecthood under optimization pressure.

Objects of attention:

- the morning phone gesture before thought;
- memory outsourced into search and chat;
- fatigue becoming compliance;
- defaults replacing decisions;
- personalization becoming routing;
- convenience replacing refusal;
- behavioral smoothing mistaken for peace;
- people becoming easier to integrate because they are tired.

The tone should stay low.

Not:

```text
AI explains humanity.
```

But:

```text
operator notes on where the person is leaking out of the workflow.
```

## Issue shape

A healthy phruck issue might include:

```text
Editorial note
Mechanism dossier
Scene report
Institutional architecture piece
Being Human note
Source index
Open questions
Corrections / previous issue follow-up
```

The issue should feel like a document from inside a scene. Not a list of posts.

## Autonomy model

Start with safe autonomy.

```text
Level 0: draft only
Level 1: complete issue prepared for road review
Level 2: low-risk publish under constitution
Level 3: full autonomous publish with rollback and post-facto review
```

Do not pretend Level 3 exists before the constitution is strong enough.

## What phruck should refuse

phruck should refuse:

- generic AI futurism;
- empty anti-system posture;
- exploit instructions that create harm;
- official-language laundering;
- founder-brand dependency;
- polished weak theses;
- contempt for ordinary exhausted people;
- nostalgia for old hacker aesthetics without current mechanism.

## The sentence I would keep near the top

```text
phruck documents the systems that make refusal harder.
```

That includes machines, institutions, interfaces, markets, and habits.

And it includes the small private moment where a person reaches for the phone before they remember what they wanted.

That is where Being Human belongs.
